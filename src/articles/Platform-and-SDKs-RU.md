## Каналы дистрибуции

Игра выходит на двух платформах:

- **WebGL** - дистрибуция через [Яндекс Игры](https://yandex.ru/games), основной канал. Полная интеграция Yandex Games SDK активна.
- **Android** - Google Play, RuStore и другие Android-сторы. Монетизация через Yandex Ads SDK; функции Yandex Games (облачные сохранения, авторизация) вне экосистемы Яндекса недоступны.

Активный набор SDK определяется флагами в `GameBuildData` на этапе сборки, а не платформенной детекцией в рантайме. Код неиспользуемых SDK не попадает в ненужный билд.

## Yandex Games SDK

SDK Яндекс Игр (плагин YG) интегрирует игру с платформой по четырём направлениям:

### Реклама

Interstitial и rewarded-реклама обслуживается через SDK. Игра вызывает YG API для показа и ждёт коллбэка перед возобновлением или выдачей награды. `LiveProgressSync.OnQuitGame()` - публичный метод без параметров, требуемый SDK, - срабатывает при закрытии вкладки браузера и инициирует синхронное финальное сохранение.

### Внутриигровые покупки

Покупки идут через платёжную систему Яндекса. SDK ведёт флоу и возвращает чек; игра консумирует покупку на сервере для разблокировки контента. Флаг `UseAddSdk` в `GameBuildData` включает/выключает этот флоу при сборке.

### Облачные сохранения

См. раздел [Облачные сохранения](#облачные-сохранения) ниже.

### Платформенные события

SDK отправляет `OnQuitGame` при выгрузке страницы браузера. `LiveProgressSync` подписывается на это событие и синхронно сбрасывает финальное сохранение до исчезновения страницы.

## Авторизация и профиль игрока

В главном меню есть **кнопка Login**. Нажатие запускает Yandex OAuth-флоу.

При успешном ответе:

1. Игра отправляет запрос профиля на сервер Яндекса.
2. Сервер отдаёт **отображаемое имя** и **URL аватара** игрока.
3. Аватар загружается асинхронно и появляется рядом с именем в UI главного меню.

При неудаче или если игрок пропускает логин, игра переходит в **гостевой режим**: только локальные `PlayerPrefs`, без синхронизации, без профиля. Геймплей полностью доступен без авторизации.

## Облачные сохранения

При включённых облачных сохранениях (`UseCloudSave = true` в `GameBuildData`) `GameProgress` синхронизируется с облачным хранилищем Яндекса в дополнение к локальному `PlayerPrefs`.

### Флоу сохранения

`SaveLoadService.SaveProgress()` сериализует `GameProgress` в JSON и пишет в `PlayerPrefs` как обычно. При активных облачных сохранениях JSON параллельно пушится в Яндекс через save API SDK.

### Флоу загрузки

При старте сессии `LoadProgressState` сначала проверяет облако. Если облачный сейв есть, он десериализуется и используется как `GameProgress`. Облачная копия всегда побеждает локальную при конфликте - прогресс с одного устройства не перезапишется старым сейвом с другого.

Если облачный запрос не удался (офлайн, нет авторизации), загрузка молча откатывается к `PlayerPrefs`.

### Автосохранение

`LiveProgressSync` автосохраняется каждые 5 секунд. При включённых облачных сохранениях каждый тик также пушится в облако.

## Yandex Ads SDK

Для Android-сборок вне экосистемы Яндекс Игр **Yandex Ads SDK** берёт на себя баннеры, interstitial и rewarded-рекламу.

Включается флагом `UseAddSdk = true` в `GameBuildData`. При `false` все код-пути Yandex Ads обходятся - та же кодовая база даёт чистый WebGL-билд для Яндекс Игр и отдельный Android-билд с рекламой, без условной компиляции в геймплейном коде.

## Конфигурация сборки

Все платформоспецифичные настройки в ScriptableObject `GameBuildData`:

| Поле | Тип | Назначение |
|---|---|---|
| `DebugConfiguration` | enum | `Development` (логи активны) или `Shipping` (логи скомпилированы) |
| `TargetPlatform` | enum | `WebGL` или `Android` |
| `UseCloudSave` | bool | Синхронизация с облачным хранилищем Яндекса |
| `UseAddSdk` | bool | Yandex Ads SDK для Android-сторов |

`FilteredEnumAttribute` на обоих enum-полях убирает сентинел `None` из дропдауна инспектора - нельзя случайно выбрать невалидное значение.

Флаг `DebugConfiguration` управляет выводом `GameLogger`. В режиме `Shipping` все блоки `#if DEVELOPMENT_BUILD || UNITY_EDITOR` компилируются в ничто - нулевые накладные расходы в продакшне.
