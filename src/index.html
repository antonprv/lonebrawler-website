О проекте

Lone Brawler — соло-проект 3D Diablo-like action RPG, разработанного в Unity с нуля до продакшн-состояния. Кодовая база включает все уровни игрового приложения: инфраструктуру, геймплей, UI и инструментарий разработчика.

В основе архитектуры — Game State Machine с payload-переходами, собственный DI-фреймворк Zenjex на базе Reflex, и кеширующий слой поверх Unity Addressables. Инъекция зависимостей работает через три прохода, гарантируя заполнение полей до вызова Awake().

C# 
public class PlayerHealth : ZenjexBehaviour,
    IHealth, IProgressReader
{
    [Zenjex] private IBuffTrackerService _buffs;
    [Zenjex] private ISoundService      _sound;

    protected override void OnAwake()
    {
        // Поля заполнены до OnAwake ✓
        _buffs.GetPlayerBuffs()
              .Subscribe(OnBuffChanged);
    }
}

Ключевые системы

Game State Machine

Единая точка управления переходами между состояниями приложения. Поддержка payload-переходов через обобщённые интерфейсы.

Zenjex — DI Framework

Собственный open-source DI-фреймворк на базе Reflex. Три прохода инъекции с дедупликацией, fluent binding API, scene-scoped sub-контейнеры.

Система баффов

Чистые C# классы — не MonoBehaviour. Три типа активации, реактивное состояние, полный пайплайн сохранения и восстановления с частичным временем.

Аудиосистема

Полностью async MusicPlayer: кросфейд, предзагрузка, Fisher-Yates shuffle. SFX с рандомизацией клипов. Реактивная привязка громкости через ISoundService.

UI — MVVM Dev Console

Оконная система на Addressables, Dev Console в полном MVVM (4 слоя, 14 команд), реактивный HUD, шкалы здоровья в мировом пространстве.

Тестирование

53 тестовых файла: EditMode юнит-тесты, PlayMode интеграционные тесты, Unity Performance Testing benchmarks. ZenjexTestBootstrap — минимальный DI для тестов.
